# Parallel Translation Assist

英文の任意の範囲に日本語訳を紐づけ、原文と訳文を並べて閲覧する個人用SPAです。React、TypeScript、Viteで構築しています。

## 主な仕様

### プロジェクト管理

- 複数の翻訳プロジェクトを作成し、サイドバーから切り替えられます。
- サイドバーの「＋」を押すと、入力画面を挟まず`New Project`というタイトルで新しいプロジェクトを作成します。
- プロジェクト名の変更とプロジェクトの削除ができます。
- プロジェクトごとに原文、対訳、ステータスを保存します。
- ステータスは次の6種類です。
  - 未着手
  - 翻訳中
  - 初稿完了
  - 修正中
  - 完了
  - 保留
- サイドバーのプロジェクト一覧には、プロジェクト名、ステータス、原文文字数、対訳数を表示します。
- サイドバーはアニメーション付きで開閉し、開いているときにサイドバー外をクリックすると閉じます。

### プロジェクト情報

- ワークスペース上部の「情報」ボタンから、タイトル、著者、出典URL、原文言語、翻訳言語を登録・更新できます。
- タイトルは必須です。ワークスペースのタイトルをクリックしても情報画面が開き、タイトル入力欄へ直接フォーカスします。
- 著者は任意です。登録するとワークスペースのタイトル下へ`by 著者名`を14pxのタイトル系セリフフォントで表示します。
- 出典URLは任意です。登録するとタイトル下へ「🔗 Source」リンクを表示し、別タブで出典を開きます。
- 原文言語の初期値は`ENGLISH`、翻訳言語の初期値は`JAPANESE`です。選択肢は`ENGLISH`、`JAPANESE`、`DEUTSCH`、`RUSSIAN`、`FRENCH`、`GREEK`、`CHINESE`、`KOREAN`です。
- 情報画面の言語選択肢にはNoto Color Emojiの国旗を表示します。選択した言語名は、編集画面の原文・訳文欄と閲覧画面の見出しへ国旗なしで反映します。
- 原文・訳文それぞれの選択言語に応じてNoto Serifファミリーを適用します。日本語はNoto Serif JP、中国語はNoto Serif SC、韓国語はNoto Serif KR、そのほかの対応言語はNoto Serifを使用します。
- 選択言語のフォントは、原文・訳文の入力欄、選択中の原文、登録済み対訳、AI下訳候補、閲覧画面へ一貫して反映します。Google Fontsのフォントファイルは、画面で実際に使用するファミリーが必要になったときにブラウザが読み込みます。

### 対訳の登録

1. 原文欄から一文、複数文、または段落を選択します。
2. 「選択範囲を翻訳」を押します。
3. 訳文欄へ日本語訳を入力し、「訳文を登録」を押します。

- `Ctrl + Enter`または`Command + Enter`でも訳文を登録できます。
- 対訳を登録した原文範囲は、原文欄で薄いオレンジ色になります。
- 「選択範囲を翻訳」を押した後は、選択した原文範囲を文字選択色と同じ色の1px枠で囲みます。
- 登録済み対訳の編集を開始した場合も原文範囲を同じ枠で囲み、原文欄を対象位置まで自動スクロールします。
- 対訳を削除すると、該当する原文範囲は通常の文字色へ戻ります。
- 登録済みの対訳は原文順に一覧表示され、編集と削除ができます。
- 「登録済みの対訳」の検索欄では、検索対象を「原文」または「訳文」から選び、「フィルター」を押すと入力文字列を含む対訳だけを表示します。検索は大文字・小文字を区別します。
- フィルター適用後も検索文字列と検索対象を変更して「フィルター」を押すことで条件を更新できます。「解除」を押すと検索欄を空にして全対訳を再表示します。検索条件は表示中だけ保持する一時的な状態で、プロジェクトデータや変更履歴には保存しません。
- 編集中は既存の訳文を入力欄へ読み込みます。「訳文を登録」を押すと既存の対訳を更新します。
- 訳文欄では次の記法を利用でき、登録済み対訳と閲覧画面で指定したスタイルとして表示します。
  - `**テキスト**`：太字
  - `_テキスト_`：下線
  - `~テキスト~`：打消し線
  - `|文字《ルビ》`または`｜文字《ルビ》`：ルビ（半角・全角の縦線に対応）
- 訳文フッター左側の「?」ボタンから、利用できる記法と表示例を確認できます。

### 未翻訳箇所への移動

- 登録済み対訳の原文範囲に含まれない連続範囲を未翻訳箇所として扱います。登録済み範囲は原文位置順に並べ、隣接または重複する範囲を結合してから、先頭、対訳間、末尾の未登録部分を抽出します。原文外の範囲は原文内へ補正します。
- 未登録部分の先頭と末尾にある空白、タブ、改行は除外し、空白類だけの範囲は未翻訳箇所として扱いません。
- 原文パネルのフッター左側に「← 前の未翻訳」と「次の未翻訳 →」を表示し、右側の既存の「選択範囲を翻訳」と並べて使用できます。狭い画面では操作が重ならないよう折り返して表示します。
- 移動すると対象範囲を原文欄で枠表示して自動スクロールし、選択した原文を訳文パネルへ表示します。空の訳文入力欄へフォーカスするため、そのまま新しい対訳を入力できます。登録済み対訳の編集モードや重複確認には入りません。
- 「次の未翻訳」は現在位置より後ろの範囲へ進み、末尾を越えると先頭へ循環します。「← 前の未翻訳」は現在位置より前の範囲へ戻り、先頭を越えると末尾へ循環します。現在位置は、対訳登録用の選択範囲、原文欄のカーソルまたは文字選択位置、原文先頭の順で決まります。
- 訳文入力欄に未保存の文字がある場合、登録済み対訳を編集中の場合、または原文更新の確認モーダルを表示している場合は、未保存内容を守るため前後の移動ボタンを無効化します。新規対訳の範囲だけが選択され、訳文入力欄が空の場合は移動できます。
- 未翻訳箇所がない場合は両方の移動ボタンを無効化し、原文パネルのフッター左側へ「← 前の未翻訳」「次の未翻訳 →」「すべて翻訳済みです」の順に表示します。対訳範囲外に空白、タブ、改行だけが残る場合も翻訳済みです。

### 訳語キーワード

- 訳文パネルのフッターで記法ヘルプの右隣にある丸囲み「+」ボタンへマウスを乗せると「キーワード追加」と表示し、押すと訳語キーワードモーダルを開きます。
- 「登録」タブで原語と訳語のセットを登録します。同じ原語は1プロジェクト内で重複登録できません。
- 「一覧」タブで登録済みセットを確認し、内容の更新と削除ができます。
- 登録した原語と単語境界を含めて完全一致する原文中の語には破線の下線を表示し、マウスを乗せると登録した訳語を表示します。英字の大文字・小文字は区別します。
- 一致範囲が重なる場合は、同じ位置から始まる最も長い原語を優先します。
- キーワード一覧はプロジェクトごとに独立し、IndexedDBへの自動保存、変更履歴、プロジェクトのインポート／エクスポートに含まれます。

### AI翻訳支援（任意機能）

- サイドバー下部の「AI翻訳支援」から、設定と外部送信に関する注意事項をモーダルで確認できます。APIキーと接続状態は全プロジェクトで共通ですが、現在のページセッション中だけ有効です。
- 利用者自身のOpenAI APIキーを入力し、「接続を確認」でブラウザから`GET https://api.openai.com/v1/models`へ直接アクセスして、APIキーが利用できるか確認します。運営者共通のAPIキーは使用しません。
- APIキーはReactのメモリ上だけに保持します。LocalStorage、SessionStorage、IndexedDB、Cookie、プロジェクト、変更履歴、エクスポートファイルには保存しません。ページを再読み込みするか「APIキーを消去」を押すと消去されます。
- APIキーは利便性との妥協として一時的にブラウザメモリへ置かれます。AI翻訳専用のOpenAI Projectと、必要なAPI権限だけを許可した制限付きキーの利用を推奨します。漏えいが疑われる場合はOpenAI Platformでキーを直ちに失効し、新しいキーを発行してください。
- 設定モーダルの「APIの使用状況はこちらで確認」から、[OpenAI PlatformのUsage画面](https://platform.openai.com/usage)を新しいタブで開けます。
- 訳文パネルのフッターでは、記法ヘルプ、「✦」AI翻訳、キーワード追加の各ボタンを横並びで表示します。「✦」からAI下訳を作成でき、APIキーが未設定の場合はホバー時に「APIキーを設定して使用」と案内します。
- AI候補は訳文入力欄を自動的に上書きせず、「AIによる下訳候補」と警告を一時表示します。「訳文欄へ反映」を押し、必要に応じて修正してから、通常の「訳文を登録」で保存します。入力中の訳文がある場合は置き換え前に確認します。
- AI候補、警告、リクエストID、使用モデル、API接続状態はプロジェクトや変更履歴へ保存しません。AI候補の反映だけでは対訳登録や変更履歴への追加を行いません。
- AI翻訳中または未反映候補の表示中は、二重送信、選択解除、未翻訳箇所への移動を無効化します。「キャンセル」は実行中の通信を中止します。プロジェクトや画面を切り替える場合は未反映候補の破棄を確認します。
- トークン数超過などのエラーが発生した場合は、選択状態と入力中の訳文を維持し、フッターの「✦」ボタンから再試行できます。

#### OpenAI APIへ送信する情報

AI翻訳支援を実行した場合、ユーザーが選択した原文、原文言語、翻訳先言語、選択範囲内に登場する訳語キーワードをOpenAI APIへ送信します。選択範囲の前後にある原文、原文全体、登録済み対訳、ほかのプロジェクトは送信しません。

- 初期実装ではトークン使用量とAPI利用料金を抑えるため、送信する原文を選択範囲だけに限定し、前後文脈を送信しません。
- 選択範囲内で一致した登録済みキーワードだけを、原語（`source`）と使用推奨訳語（`translated`）のセットとして重複排除して送信し、最大100件とします。システムプロンプトでも、このセットを原文の意味や文法と矛盾しない限り訳文へ使用するよう指示します。選択範囲外にだけ登場するキーワードは送信しません。
- 前後文脈がないため、訳語、主語、指示語、固有名詞、語義、文体などの判断精度が下がる場合があります。文脈不足の場合はAI候補とともに警告を表示します。必要な場合は選択範囲を広げて再実行できます。
- AI翻訳では原文の段落の切れ目を維持します。段落内の単なる改行はそのまま残さず、訳文として自然につながる文章にします。
- プロジェクトID、タイトル、著者、出典URL、変更履歴、AI候補の過去履歴、APIキー以外のブラウザ保存データは送信しません。APIキーはJSON本文やURLではなく、Bearer認証ヘッダーでOpenAI APIへ直接送信します。
- 利用料金は入力したAPIキーの所有者に発生します。

#### OpenAI APIとの直接通信とエラー

- このアプリはAPサーバーや`/api/`エンドポイントを持ちません。接続確認はOpenAIのModels API、翻訳はResponses APIへブラウザの`fetch`から直接アクセスします。
- 翻訳モデル、システム指示、出力スキーマはアプリ内で固定し、利用者が任意のAPI URL、外部URL、ツール定義を指定する機能は設けません。
- `sourceText`は1～12,000文字、訳語は最大100件・各200文字に制限します。`contextBefore`、`contextAfter`、`fullSource`など選択範囲外の情報はリクエストへ追加しません。
- 中継サーバーを使用しないため、APIキーは利用者のブラウザからOpenAI APIへ直接送信され、ブラウザの開発者ツールから確認できる状態になります。共有端末や信頼できない環境では使用せず、AI翻訳専用の制限付きキーを使用してください。
- APIキー無効、権限不足、利用上限、レート制限、タイムアウト、通信失敗、不正なAI応答を個別に表示します。エラーが発生しても、現在の原文選択、入力中の訳文、登録済み対訳、変更履歴は変更しません。
- 開発・テストではOpenAI APIを完全にモックし、実APIキーや実際のOpenAI APIを使用しません。

開発時は`npm run dev`、ビルド結果の確認時は`npm run preview`を使用します。本番ビルドは静的ファイルとして配信でき、AI翻訳のためのNodeサーバーや環境変数は不要です。OpenAI APIへの接続を許可するContent Security Policyを維持してください。HTMLのmeta CSPには、metaでは無効となる`frame-ancestors`を指定しません。フレーム埋め込みを禁止する場合は、配信環境のHTTPレスポンスヘッダーで`Content-Security-Policy: frame-ancestors 'none'`を設定してください。

### 登録済み範囲を含む選択

選択した原文範囲が登録済みの対訳と重なる場合、次の規則で処理します。

- 選択範囲が登録済み対訳の原文範囲と完全一致する場合
  - その対訳の編集モードへ自動的に移行します。
- 完全一致せず、一部または全部が重なる場合
  - 確認モーダルを表示します。
  - 「キャンセル」：登録済み対訳を変更せず、操作を中止します。
  - 「破棄して続ける」：重なる対訳を削除し、選択範囲の新しい訳文入力を開始します。
  - 「訳文を保持」：重なる対訳の訳文を原文順に空行で結合し、新しい訳文入力欄へ引き継ぎます。

対訳の編集中に原文を選択し直した場合は、編集対象の原文範囲を更新します。ほかの対訳と重なる場合は同じ確認モーダルを表示し、破棄または訳文の結合を選択できます。

### 登録済み対訳がある原文の編集

原文そのものを編集した場合、変更箇所と登録済み対訳の位置関係に応じて処理します。

- 変更箇所が登録済み対訳の範囲外の場合
  - 確認は表示しません。
  - 原文を更新し、登録済み対訳を維持します。
  - 変更箇所より後ろにある対訳の位置を自動的に補正します。
- 変更箇所が登録済み対訳の原文範囲に影響する場合
  - 「この原文の更新は登録済みの対訳に影響します」という確認モーダルを表示します。
  - 「キャンセル」：原文を更新せず、登録済み対訳も維持します。
  - 「破棄して続ける」：影響を受ける対訳だけを削除し、原文を更新します。影響しない対訳は維持します。
  - 「訳文を保持」：訳文を維持し、対訳側の原文範囲と原文文字列を更新後の内容へ追従させます。

「訳文を保持」を選択した後は入力直後のカーソル位置へ戻り、原文欄からフォーカスを外すまで、同じ対訳に対する連続入力を同じ方針で処理します。入力が800ミリ秒以上途切れても、フォーカスが残っている間は確認モーダルを再表示しません。

### 並列閲覧

- 閲覧モードでは原文と日本語訳を左右に並べて表示します。
- 閲覧画面の上部には「原文と訳文」という補助見出し、プロジェクトタイトル、登録されている場合は著者と出典リンクを編集画面と同じスタイルで表示します。
- 閲覧画面のタイトルは参照専用で、クリックしてもプロジェクト情報画面は開きません。
- 未訳部分も原文の順序を維持して表示します。
- ブラウザの印刷機能では、アプリケーションヘッダーと訳文件数の説明を印刷対象から除外します。
- スマートフォンでは縦方向に並べて表示します。

### 統計

- ワークスペース上部の「統計」ボタンから、現在のプロジェクトの進捗をモーダルで確認できます。
- 原文単語数、原文文字数、対訳件数、翻訳済み割合、翻訳済み／未翻訳単語数、訳文合計文字数、プロジェクトの最終更新日時を表示します。
- 翻訳済み割合は、原文単語数に対する翻訳済み単語数の割合です。すべての単語が登録済み対訳の原文範囲に含まれる場合は、範囲外に改行や空白が残っていても100%と表示します。
- 単語は文字・数字のまとまりを基準に数え、アポストロフィやハイフンでつながった語は1語として扱います。単語全体が登録済み対訳の原文範囲に含まれる場合に翻訳済みとして数えます。
- 翻訳済み割合は進捗リングとバーで、単語数などは内訳カードで表示します。

### データ保存

- プロジェクト、対訳、訳語キーワード、変更履歴はブラウザのIndexedDBへ自動保存します。
- 通常のプロジェクトデータはブラウザ内へ保存されます。AI翻訳支援を実行した場合に限り、選択した原文、原文言語、翻訳先言語、選択範囲内に登場する訳語キーワードをブラウザからOpenAI APIへ直接送信します。選択範囲の前後にある原文やプロジェクト全体は送信しません。入力されたOpenAI APIキーはセッション中のブラウザメモリだけに保持し、永続保存しません。
- データはブラウザと端末ごとに独立しており、別のブラウザや端末とは同期されません。
- ブラウザのサイトデータを削除すると保存内容も削除されます。
- 旧バージョンがLocalStorageへ保存したデータは移行せず、新しい初期プロジェクトから開始します。

### 変更履歴・戻す／進む

- ワークスペース上部の「戻す」と「進む」で、プロジェクト内容の変更を取り消したり、やり直したりできます。
- タイトル、著者、出典URL、原文言語、翻訳言語、ステータス、原文、登録済み対訳、訳語キーワードをまとめて1つの状態として復元します。
- 変更履歴はプロジェクトごとに独立しており、過去状態と未来状態を合わせて最大25件保持します。
- 履歴はIndexedDBへ保存されるため、ブラウザを再読み込みした後も「戻す」と「進む」を利用できます。
- 「戻す」の後に「進む」を使わず新しい変更を行うと、その時点で保持していた未来状態は破棄されます。
- 原文欄への連続入力は1つの変更として扱い、最後の入力から800ミリ秒経過するか、原文欄からフォーカスが外れると次の変更単位になります。
- 原文の選択範囲、入力途中の訳文、編集モード、表示切り替えなどの一時的なUI状態は履歴に含まれません。

### インポート

ヘッダーの「インポート」から、原文またはプロジェクト全体を読み込めます。ファイル内容はブラウザ内で処理され、外部へ送信されません。

#### プロジェクトをインポート

- Parallel Translation Assistがエクスポートしたバージョン付き`.json`ファイルに対応します。
- タイトル、著者、出典URL、原文言語、翻訳言語、ステータス、原文、登録済み対訳、訳語キーワードを復元します。インポートの実行日時がプロジェクトの新しい最終更新日時になります。
- 現在のプロジェクトに原文または対訳がある場合は、すべて上書きされることを確認するモーダルを表示します。
  - 「キャンセル」：インポートを中止します。
  - 「続ける」：現在のプロジェクト内容をインポートファイルの内容へ置き換えます。
- JSONが破損している、形式が異なる、または未対応バージョンの場合はインポートしません。

#### 原文をインポート

- `.txt`と`.md`ファイルに対応します。
- 現在の原文が空の場合は、そのまま原文欄へ読み込みます。
- すでに原文がある場合は確認モーダルを表示します。
  - 「キャンセル」：インポートを中止します。
  - 「上書き」：現在の原文を置き換え、登録済みの対訳をすべて削除します。
  - 「続きとして追加」：現在の原文の末尾へ改行して追加し、登録済みの対訳を維持します。

### エクスポート

ヘッダーの「エクスポート」から、現在選択しているプロジェクトを次の形式で保存できます。

- プロジェクトをエクスポート
  - タイトル、著者、出典URL、原文言語、翻訳言語、ステータス、原文、対訳とその原文位置、最終更新日時を含む`.pta.json`ファイルを保存します。
  - 保存したファイルは「プロジェクトをインポート」で復元できます。
  - 変更履歴は`.pta.json`へ含めません。インポート時も現在のプロジェクト状態だけを扱います。
- 訳文をエクスポート
  - 1行目にプロジェクトタイトル、2行目に空行、その後に登録済み訳文を原文順・1対訳1行で並べた`.txt`ファイルを保存します。
- 対訳をテキストでエクスポート
  - 1行目にプロジェクトタイトル、その後に原文と訳文の組を空行区切りで並べた`.txt`ファイルを保存します。
  - 対訳が未登録の原文部分も出力し、訳文には`—`を表示します。

### `.pta.json`プロジェクトファイル形式

プロジェクトファイルはUTF-8のJSONです。現在の形式バージョンは`1`で、次の構造を使用します。

```json
{
  "format": "parallel-translation-assist",
  "version": 1,
  "exportedAt": "2026-07-16T12:34:56.789Z",
  "project": {
    "id": "project-1",
    "title": "Alice / Chapter 1",
    "author": "Lewis Carroll",
    "sourceUrl": "https://example.com/alice",
    "originalLanguage": "ENGLISH",
    "translatedLanguage": "JAPANESE",
    "status": "翻訳中",
    "source": "Alice was beginning to get very tired.",
    "updatedAt": "2026-07-16T12:30:00.000Z",
    "translations": [
      {
        "id": "translation-1",
        "start": 0,
        "end": 38,
        "source": "Alice was beginning to get very tired.",
        "translated": "アリスはとても退屈しはじめていました。"
      }
    ],
    "keywords": [
      {
        "id": "keyword-1",
        "source": "tired",
        "translated": "退屈した"
      }
    ]
  }
}
```

| フィールド | 内容 |
| --- | --- |
| `format` | ファイル種別を識別する固定値`parallel-translation-assist` |
| `version` | ファイル形式のバージョン。現在は`1` |
| `exportedAt` | エクスポート日時を表すISO 8601形式の文字列 |
| `project.id` | エクスポート元のプロジェクトID。インポート時は現在選択中のプロジェクトIDを維持します |
| `project.title` | プロジェクトタイトル |
| `project.author` | 原文の著者。未登録の場合は空文字列 |
| `project.sourceUrl` | 原文の出典URL。未登録の場合は空文字列 |
| `project.originalLanguage` | 原文言語。対応する8言語のいずれか |
| `project.translatedLanguage` | 翻訳言語。対応する8言語のいずれか |
| `project.status` | `未着手`、`翻訳中`、`初稿完了`、`修正中`、`完了`、`保留`のいずれか |
| `project.source` | 原文全体 |
| `project.updatedAt` | プロジェクト内容の最終更新日時を表すISO 8601形式の文字列 |
| `project.translations` | 登録済み対訳の配列 |
| `project.translations[].id` | 対訳を識別するID |
| `project.translations[].start` | 原文全体における対訳範囲の開始位置（開始位置を含む） |
| `project.translations[].end` | 原文全体における対訳範囲の終了位置（終了位置を含まない） |
| `project.translations[].source` | 対訳として登録した原文 |
| `project.translations[].translated` | 登録した訳文 |
| `project.keywords` | プロジェクト固有の訳語キーワード配列 |
| `project.keywords[].id` | キーワードを識別するID |
| `project.keywords[].source` | 原文で照合する原語 |
| `project.keywords[].translated` | ホバー時に表示する訳語 |

`start`と`end`はJavaScript文字列のインデックス（UTF-16コード単位）です。通常は`project.source.slice(start, end)`が対訳の`source`に対応します。

インポート時は`format`と`version`が現在の値に一致し、プロジェクトの必須フィールドの型、ステータス、言語、対訳範囲、訳語キーワードが有効なファイルだけを受け付けます。プロジェクト情報や訳語キーワードがない旧形式のバージョン1ファイルは、著者と出典URLを空欄、原文言語を`ENGLISH`、翻訳言語を`JAPANESE`、訳語キーワードを空配列として読み込みます。`project.updatedAt`がない場合は`exportedAt`を最終更新日時として使用します。未対応バージョンは読み込まず、エラーを表示します。将来、互換性を壊す形式変更を行う場合は`version`を更新し、旧バージョンの移行処理を実装するか、未対応として明示的に拒否します。

## 開発

```sh
npm install
npm run dev
```

### テストとビルド

```sh
npm test -- --runInBand
npm run test:e2e
npm run build
```

- 単体テストにはJest、ts-jest、Testing Libraryを使用しています。
- ブラウザ上の主要な利用フローはPlaywright E2Eテストで確認します。初回は`npx playwright install chromium`でテスト用ブラウザをインストールしてください。
- Huskyのコミット前フックがJestとPlaywright E2Eテストを実行し、いずれかが失敗した場合はコミットを中止します。
- GitHub ActionsでもJest、Playwright E2Eテスト、本番ビルドがすべて成功した場合だけGitHub Pagesへデプロイします。
- 機能や仕様を変更した場合は、テストとこのREADMEを同時に更新してください。

## GitHub Pages

`main`ブランチへのpushでGitHub Actionsがビルドし、GitHub Pagesへ公開します。リポジトリの **Settings → Pages → Build and deployment → Source** を **GitHub Actions** に設定してください。

`robots.txt`と`noindex`メタタグで検索エンジンへの掲載を拒否しています。ただし、GitHub Pagesにはアクセス制限がないため、URLを知っている人はページを閲覧できます。
