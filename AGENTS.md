# Development Instructions

このリポジトリで変更を行うすべてのエージェントは、以下を必ず守ること。

## 設計

- `src/App.tsx` に表示・ドメインロジックを集中させない。
- 再利用可能な表示要素は `src/components/`、状態管理は `src/hooks/`、純粋なドメインロジックは `src/domain/`、永続化など外部境界は `src/services/` に配置する。
- 共通の型は `src/types.ts` に定義する。
- Reactコンポーネントは単一責任を保ち、propsを明示的に型付けする。

## 仕様書

- `README.md`を現行仕様の基準文書として維持する。
- 機能追加、挙動変更、表示上の仕様変更、データ保存方式の変更を行う場合は、同じ変更内で`README.md`の該当箇所を必ず更新する。
- 実装と`README.md`の記載に差異がある状態でコミットしない。

## テスト

- 挙動を追加・変更した場合は、対応するJest単体テストを追加または更新する。
- テストは対象に近い `*.test.ts` または `*.test.tsx` に配置し、`ts-jest` で実行する。
- コミット前に必ず `npm test -- --runInBand` を実行する。
- テスト失敗時はコミットせず、失敗原因を特定してコードまたはテストを修正し、全テストが成功するまで再実行する。
- テストを削除・スキップして失敗を回避しない。

## 検証とコミット

- コミット前に `npm run build` も実行し、型検査と本番ビルドを確認する。
- `.husky/pre-commit` を維持し、Jestが成功した場合だけコミットを許可する。
- 生成物（`dist/`、`*.tsbuildinfo`）や `node_modules/` はコミットしない。
- コミットには、今回の依頼に関係する変更だけを含める。
